# Knowledge-Graph Protocol — Repository Intelligence Graph (RIG)

Builds and maintains a structured map of codebase architecture, dependencies,
and decisions across all Platform repos.

## Trigger
- "init" on new or existing project
- Story completion (add story→file links)
- Architecture changes (add/update nodes)
- User says "graph", "architecture", "map"

## Structure

Node Types: repo, module, service, contract, decision
Edge Types: depends_on, calls, shares, decided_by

Each node has: id, type, language, dependencies[], owner, coverage%.

## Vulnerability Overlay (v1.0)

Nodes include security/dependency metadata:
- vulnerability_status: clean | warn | vulnerable | critical
- outdated_deps: array of {name, current, latest, severity, cve}
- last_audit: ISO timestamp
- coverage: percentage

Visualization color-codes nodes: green/yellow/red/pulsing-red.
Data source: .ai/dependency-status.json (generated by CI workflow).

## Output
Write to .ai/knowledge-graph.json

## Rules
- Graph is the single source of truth for architecture
- Never delete nodes (mark as deprecated)
- Always update on story completion
- Cross-repo edges require init --repos
